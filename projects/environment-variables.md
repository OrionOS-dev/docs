# Environment Variables

Complete reference for configuring OrionOS through environment variables.

## Overview

OrionOS uses environment variables for configuration. Store these in a `.env` file in your project root.

## Required Variables

### OrionOS API

```bash
# OrionOS API Key (required for API access)
ORION_API_KEY=your_orionos_api_key_here
```

Get your API key from [orionos.dev/settings/api-keys](https://orionos.dev/settings/api-keys)

## Loading Environment Variables

### Node.js (dotenv)

```javascript
import dotenv from 'dotenv';

// Load .env file
dotenv.config();

// Load specific file
dotenv.config({ path: '.env.production' });

// Access variables
const apiKey = process.env.ORION_API_KEY;
```

### Docker

```dockerfile
# In Dockerfile
ENV NODE_ENV=production

# Or in docker-compose.yml
environment:
  - ORION_API_KEY=${ORION_API_KEY}
  - NODE_ENV=production
```

### Kubernetes

```yaml
# ConfigMap
apiVersion: v1
kind: ConfigMap
metadata:
  name: orionos-config
data:
  NODE_ENV: "production"
  PORT: "3000"

# Secret
apiVersion: v1
kind: Secret
metadata:
  name: orionos-secrets
type: Opaque
data:
  ORION_API_KEY: base64encodedvalue
```

## Security Best Practices

1. **Never commit .env files** - Add to `.gitignore`
2. **Use different keys per environment** - Dev, staging, production
3. **Rotate keys regularly** - Every 90 days minimum
4. **Limit key permissions** - Use least privilege principle
5. **Monitor key usage** - Track and alert on unusual activity
6. **Use secret management** - AWS Secrets Manager, HashiCorp Vault
7. **Encrypt sensitive values** - In production environments

## Validation

Validate environment variables on startup:

```typescript
import { z } from 'zod';

const envSchema = z.object({
  ORION_API_KEY: z.string().min(1),
  PORT: z.string().transform(Number).pipe(z.number().int().positive()),
  NODE_ENV: z.enum(['development', 'production', 'test']),
  DATABASE_URL: z.string().url(),
});

const env = envSchema.parse(process.env);
```

## Troubleshooting

### Variable Not Found

```bash
# Check if variable is set
echo $ORION_API_KEY

# Check .env file is loaded
node -e "require('dotenv').config(); console.log(process.env.ORION_API_KEY)"
```

### Wrong Value

```bash
# Check for trailing spaces
cat .env | grep ORION_API_KEY | od -c

# Check for quotes
# Wrong: ORION_API_KEY="value"
# Right: ORION_API_KEY=value
```
